# 红苹果AI用户系统 v1

功能：
- 用户注册/登录
- 用户独立添加设备
- 用户独立添加模型配置
- 用户独立设置角色 Prompt
- 为设备绑定模型、角色、固件
- 上传固件 bin 文件
- 本地一键烧录 ESP32/ESP32-S3 固件
- 聊天测试调用当前默认模型和默认角色

运行：
1. 解压
2. 双击 run_windows.bat
3. 打开 http://127.0.0.1:8000
4. 默认账号：admin / redapple123

重要：
一键烧录只能在连接开发板的本地电脑运行。云服务器无法直接访问用户电脑 USB 串口。


## v1.1 登录修复

修复内容：
- 登录页不再默认填入 admin / redapple123；
- 注册成功后自动回到登录页，并填入刚注册的用户名；
- 新注册用户会自动生成默认角色和默认 Ollama 模型配置；
- 总览页显示当前用户 ID，便于确认当前登录的是哪个用户。

注意：
如果你浏览器已经登录了 admin，需要先点击左侧底部“退出登录”，再用新用户登录。


## v1.2 角色设备表现设置

角色设置新增：
- 声音设置：温柔女声、活泼女声、沉稳男声、儿童音色、机器人音色、自定义声音
- 眼睛设置：自然眨眼、微笑眼、好奇眼、安静聆听、思考动画、睡眠模式
- 语言设置：中文、英文、中英双语、儿童中文、简洁中文

这些配置会保存到角色表，并在调用模型时附加到角色提示词中。
设备管理中绑定角色时也会显示角色对应的声音和语言，方便选择。

## v1.4 简化模型配置

调整内容：
- 模型配置页不再让用户填写 API 地址、模型名、API Key；
- 用户只需要选择模型；
- 后台根据模型选择自动写入 API 地址、模型名和默认参数；
- 本地 Ollama 默认无需 API Key；
- DeepSeek 等云端模型建议由管理员在环境变量中统一配置 API Key。

当前内置模型：
- 本地 Ollama - Qwen2.5 1.5B
- 本地 Ollama - Qwen2.5 0.5B
- DeepSeek - deepseek-chat
- DeepSeek - deepseek-reasoner


## v1.5 界面简化与角色页修复

调整内容：

- 模型配置页只列出可选择模型，去掉复杂 API 说明；
- 角色设置页改为卡片式显示，避免表格挤压错乱；
- “设为默认角色”复选框显示修复；
- “自定义声音”增加自定义声音名称输入框，并真正保存到角色配置中。


## v1.6 固件流程调整

固件烧录页调整为两个核心功能：
1. 根据设置生成固件配置
   - 读取设备、模型、角色、声音、眼睛、语言、Token、WebSocket 地址
   - 生成 JSON 配置文件
   - 保存到 uploads/configs 目录
2. 一键烧录
   - 选择设备、固件包、串口
   - 调用 esptool 烧录
   - 烧录前会提示是否已有最近生成的设备配置

说明：
当前版本生成的是配置 JSON 文件。真实产品中可以进一步将该配置嵌入 NVS 分区、LittleFS 分区，或让设备首次启动后通过 Token 从云端拉取配置。
